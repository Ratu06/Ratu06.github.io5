Slide 1: Review Sesi 4-Polimorfisme & Abstraksi

Polimorfisme: Objek subclass sebagai superclass

Abstract class: tidak bisa diinstansiasi, bisa punya method abstract/concrete

Interface: implicit di Dart, gunakan implements

Contoh cepat:

abstract class Bentuk {
  double hitungLuas();
}
class Persegi implements Bentuk {
  double sisi;
  Persegi(this.sisi);
  @override double hitungLuas() => sisi * sisi;
}

Slide 2: Multiple Inheritance dengan mixin

Mixin di Dart digunakan untuk menggabungkan fungsi dari beberapa class tanpa harus memakai inheritance yang rumit.

Multiple inheritance adalah pewarisan dari lebih dari satu superclass sekaligus.

Bahasa seperti C++, Python, Scala, Perl mendukung multiple inheritance, sedangkan Java, C#, Kotlin hanya mendukung single inheritance.

Tujuan multiple inheritance adalah menggabungkan berbagai perilaku/properti dalam satu objek tanpa duplikasi kode.

mixin CanFly {
  bool _isFlying = false;
  bool get isFlying => _isFlying;
  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print('Started flying!');
    } else {
      print('Already flying!');
    }
  }
  void land() {
    if (_isFlying) {
      _isFlying = false;
      print('Landed!');
    } else {
      print('Already on the ground!');
    }
  }
}
mixin CanSwim {
  bool _isSwimming = false;
  bool get isSwimming => _isSwimming;
  void swim() {
    if (!_isSwimming) {
      _isSwimming = true;
      print('Started swimming!');
    } else {
      print('Already swimming!');
    }
  }
  void stopSwim() {
    if (_isSwimming) {
      _isSwimming = false;
      print('Stop swimming!');
    } else {
      print('Already on stop swimming!');
    }
  }
}

class Bird with CanFly {}
class Duck extends Bird with CanSwim {}
class Fish with CanSwim {}
class Airplane with CanFly {}

void main() {
  Bird bird = Bird();
  bird.fly(); bird.fly(); bird.land(); bird.land();
  print('Is Bird flying? : ${bird.isFlying}');
  print('----------------');
  Duck duck = Duck();
  duck.fly(); duck.swim(); duck.swim(); duck.stopSwim(); duck.stopSwim();
  print('Is Duck swimming? : ${duck.isSwimming}');
  print('----------------');

  fish.swim(); fish.stopSwim();
  print('Is Fish Swimming? : ${fish.isSwimming}');

Slide 5: Pengantar Class Modifiers

Apa itu? Keyword untuk mengontrol bagaimana class dapat digunakan.

Tujuan: Kontrol inheritance/implementasi, jaminan keamanan tipe, desain API yang lebih ekspresif.

Macam-macam: sealed, base, final, interface (Dart 3.0+)

Slide 6: Sealed class

Fitur: Tidak bisa di-extend atau di-implement dari luar library. Membatasi hierarki tipe yang diketahui saat compile-time. Perfect untuk pattern matching.


//Nama: Ratu Klarisa Rahadianti
//Nim: 251410010
//Kelas: SI2A

sealed class ApiResponse {
  const ApiResponse();
}
class Success extends ApiResponse {
  final String message;
  const Success(this.message);
}
class ErrorResponse extends ApiResponse {
  final String errorMessage;
  const ErrorResponse(this.errorMessage);
}
void handle(ApiResponse response) {
  if (response is Success) {
    print("Success: ${response.message}");
  } else if (response is ErrorResponse) {
    print("Error: ${response.errorMessage}");
  }
}
void main() {
  handle(const Success("Data berhasil"));
  handle(const ErrorResponse("Terjadi kesalahan"));
}

Slide 7: Keuntungan Sealed class (Pattern Matching)

//Nama: Ratu Klarisa Rahadianti
//Nim: 251410010
//Kelas: SI2A

void handleResponse(ApiResponse response) {
  switch (response) {
    case Success():
      print("Success: ${(response as Success).message}");
      break;
    case Error():
      print("Error: ${(response as Error).errorMessage}");
      break;
  }
}
void main() {
  handleResponse(Success("Data berhasil"));
  handleResponse(Error("Terjadi kesalahan"));
}
sealed class ApiResponse {}
class Success extends ApiResponse {
  final String message;
  Success(this.message);
}
class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

Slide 8: Base class

Fitur: Bisa di-extend, tapi tidak bisa di-implement. Memastikan pewarisan dengan extends, bukan implements.

    print("Hewan bersuara...");
  }

  void speak() {
    print("woof woff!");
  }

  Animal myDog = Dog();
  myDog.speak(); // Output: woof woff!
}

Slide 10: Final Class

Fitur: Tidak bisa di-extends atau di-implements di luar library. Berguna untuk kelas yang ingin ditutup.

//Nama: Ratu Klarisa Rahadianti
//Nim: 251410010
//Kelas: SI2A

final class Settings {
  final String theme;
  final int fontSize;
  Settings({required this.theme, required this.fontSize});
  void showSettings() {
    print("Theme: $theme, Font Size: $fontSize");
  }
}

class CustomSettings {
  final Settings settings;
  CustomSettings(String theme, int fontSize)
      : settings = Settings(theme: theme, fontSize: fontSize);
  void showSettings() {
    settings.showSettings();
  }
}

void main() {
  Settings settings = Settings(theme: "Dark", fontSize: 14);
  settings.showSettings();
  CustomSettings custom = CustomSettings("Dark", 14);
  custom.showSettings();
}

Slide 11: Perbedaan Class Modifiers

Sealed: Hanya bisa di-extend di dalam file yang sama; tidak bisa di-extend atau di-implement di luar library.

base: Bisa di-extend di luar library, tetapi tidak bisa di-implement (hanya mendukung pewarisan).

final: Bisa di-extend atau di-implement di dalam file yang sama, namun keduanya dilarang dilakukan di luar library.

default: Tidak memiliki batasan; bebas untuk di-extend maupun di-implement baik di dalam maupun di luar library.

Slide 12: Praktik 1 - Sealed class dengan pattern matching

//Nama: Ratu Klarisa Rahadianti
//Nim: 251410010
//Kelas: SI2A

sealed class Status { const Status(); }
class Loading extends Status { const Loading(); }
class Loaded extends Status { final String data; const Loaded(this.data); }
class Failed extends Status { final String error; const Failed(this.error); }

void handleStatus(Status status) {
  switch (status) {
    case Loading(): print('Loading...');
    case Loaded(:final data): print('Data: $data');
    case Failed(:final error): print('Error: $error');
  }
}

void main() {
  handleStatus(Loading());
  handleStatus(Loaded('Koneksi gagal'));
  handleStatus(Failed('Data tidak ditemukan'));
}

sealed class AppState {}
class LoadingState extends AppState {}
class SuccessState extends AppState { final List items; SuccessState(this.items); }
class ErrorState extends AppState { final String message; ErrorState(this.message); }

void updateUI(AppState state) {
  switch (state) {
    case LoadingState(): print('⏳ Loading...');
    case SuccessState(:final items): print('✅ Success! Items: $items');
    case ErrorState(:final message): print('❌ Error: $message');
  }
}

void main() {
  updateUI(LoadingState());
  updateUI(SuccessState(['Dart', 'Flutter']));
  updateUI(ErrorState('Koneksi gagal'));
}

Ringkasan & Quick Guide

Sealed: Hierarki tertutup & aman untuk pattern matching.

Base: Hanya bisa di-extend (tutup akses implement).

Final: Segel total (tidak bisa di-extend atau di-implement).

🔹 Sealed → State atau Result terbatas. 🔹 Base → Framework / Base Class. 🔹 Final → Utility / Constant Class.
