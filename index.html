Slide 2: Multiple Inheritance dengan mixin

Mixin di Dart digunakan untuk menggabungkan fungsi dari beberapa class tanpa harus memakai inheritance yang rumit.

Multiple inheritance adalah pewarisan dari lebih dari satu superclass sekaligus.

Bahasa seperti C++, Python, Scala, Perl mendukung multiple inheritance, sedangkan Java, C#, Kotlin hanya mendukung single inheritance.

Tujuan multiple inheritance adalah menggabungkan berbagai perilaku/properti dalam satu objek tanpa duplikasi kode.

// Mixin dengan Properti (Opsional)
mixin CanFly {
  bool _isFlying = false; // Properti (private)
  bool get isFlying => _isFlying; // Getter
  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print('Started flying!');
    } else {
      print('Already flying!');
    }
  }

  void land() {
    if (_isFlying) {
      _isFlying = false;
      print('Landed!');
    } else {
      print('Already on the ground!');
    }
  }
}
mixin CanSwim {
  bool _isSwimming = false; // Properti (private)
  bool get isSwimming => _isSwimming; // Getter
  void swim() {
    if (!_isSwimming) {
      _isSwimming = true;
      print('Started swimming!');
    } else {
      print('Already swimming!');
    }
  }

  void stopSwim() {
    if (_isSwimming) {
      _isSwimming = false;
      print('Stop swimming!');
    } else {
      print('Already on stop swimming!');
    }
  }
}

class Bird with CanFly {}

class Duck extends Bird with CanSwim {}

class Fish with CanSwim {}

Slide 6 Sealed class

Fitur

Tidak bisa id-ectend atau di-implement dar luar library

Membatasi hierarki tipe yang diketahui saat compile-time

Perfect untuk pattern matching

sealed class ApiResponse {
  const ApiResponse();
}
// ================= SUCCESS =================
class Success extends ApiResponse {
  final String message;
  const Success(this.message);
}
// ================= ERROR =================
class ErrorResponse extends ApiResponse {
  final String errorMessage;
  const ErrorResponse(this.errorMessage);
}
void handle(ApiResponse response) {
  if (response is Success) {
    print("Success: ${response.message}");
  } else if (response is ErrorResponse) {
    print("Error: ${response.errorMessage}");
  }
}

void main() {
  handle(const Success("Data berhasil"));
  handle(const ErrorResponse("Terjadi kesalahan"));
}

class Success extends ApiResponse {
  final String message;

}
class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

final class Settings {
  final String theme;
  final int fontSize;

  Settings({required this.theme, required this.fontSize});

  void showSettings() {
    print("Theme: $theme, Font Size: $fontSize");
  }
}

// ❌ Tidak boleh extends Settings karena final
// jadi kita pakai class biasa yang berdiri sendiri
class CustomSettings {
  final Settings settings;

  CustomSettings(String theme, int fontSize)
      : settings = Settings(theme: theme, fontSize: fontSize);

  void showSettings() {
    settings.showSettings();
  }
}

void main() {
  Settings settings = Settings(theme: "Dark", fontSize: 14);
  settings.showSettings();

  CustomSettings custom = CustomSettings("Dark", 14);
  custom.showSettings();
}

Slide 11 Perbedaan Class Modifiers

Sealed: Hanya bisa di-extend di dalam file yang sama; tidak bisa di-extend atau di-implement di luar library.

base: Bisa di-extend di luar library, tetapi tidak bisa di-implement (hanya mendukung pewarisan).

final: Bisa di-extend atau di-implement di dalam file yang sama, namun keduanya dilarang dilakukan di luar library.

default:Tidak memiliki batasan; bebas untuk di-extend maupun di-implement baik di dalam maupun di luar library.

Slide 12 P: Praktik 1-Sealed class dengan pattern matching

Sealed class memungkinkan semua subclass diketahui dalam satu file sehingga dengan pattern matching setiap kondisi bisa diperiksa dan ditangani dengan jelas.

//Nama: Ratu Klarisa Rahadianti
//Nim: 251410010
//Kelas: SI2A

sealed class Status {
  const Status();
}

class Loading extends Status {
  const Loading();
}

class Loaded extends Status {
  final String data;
  const Loaded(this.data);
}

class Failed extends Status {
  final String error;
  const Failed(this.error);
}

void handleStatus(Status status) {
  switch (status) {
    case Loading():
      print('Loading...');
    case Loaded(:final data):
      print('Data: $data');
    case Failed(:final error):
      print('Error: $error');
  }
}

void main() {
  print('=' * 60);
  print('DEMONSTRASI SEALED CLASS + PATTERN MATCHING');
  print('=' * 60);

  // Simulasi berbagai state
  final states = [
    Loading(),
    Loaded('Koneksi gagal'),
    Failed('Data tidak ditemukan'),
  ];

  for (int i = 0; i < states.length; i++) {
    print('\n[${i + 1}] State: ${states[i].runtimeType}');
    print('-' * 60);
    handleStatus(states[i]);
  }

  print('\n' + '=' * 60);
  print('KEUNGGULAN SEALED CLASS:');
  print('=' * 60);
  print('✅ Exhaustive switch: Compiler memastikan semua kasus tertangani');
  print('✅ Type-safe: Tidak ada runtime errors karena missing case');
  print('✅ Pattern matching: Destructuring field langsung di switch');
  print('✅ Clear hierarchy: Hanya subclass yang didefinisikan yang bisa extend');
  print('=' * 60);
}

Slide 13: Studi Kasus: Sistem State Management

State Management adalah cara mengatur kondisi aplikasi (loading, success, error) dengan sealed class agar tiap state jelas.

Fitur:

Mengelola kondisi & alur program

Kenapa digunakan:

Lebih terstruktur dan Mencegah error tidak ter-handle

Batasan:

Bisa jadi kompleks jika state terlalu banyak

//Nama: Ratu Klarisa Rahadianti
//Nim: 251410010
//Kelas: SI2A

sealed class AppState {}

class LoadingState extends AppState {}

class SuccessState extends AppState {
  final List items;
  SuccessState(this.items);
}

class ErrorState extends AppState {
  final String message;
  ErrorState(this.message);
}

// Handler yang aman dengan exhaustive switch
void updateUI(AppState state) {
  switch (state) {
    case LoadingState():
      showLoading();
      break;

    case SuccessState(:final items):
      showItems(items);
      break;

    case ErrorState(:final message):
      showError(message);
      break;
  }
}

void showLoading() {
  print('⏳ Loading...');
}

void showItems(List items) {
  print('✅ Success! Items:');
  for (int i = 0; i < items.length; i++) {
    print('   ${i + 1}. ${items[i]}');
  }
}

void showError(String message) {
  print('❌ Error: $message');
}

void main() {
  print('=' * 60);
  print('DEMONSTRASI SEALED CLASS + PATTERN MATCHING');
  print('=' * 60);

  final states = [
    LoadingState(),
    SuccessState(['Item 1', 'Item 2', 'Item 3']),
    ErrorState('Koneksi gagal'),
    SuccessState(['Dart', 'Flutter', 'Pattern Matching']),
    ErrorState('Data tidak ditemukan'),
  ];

  for (int i = 0; i < states.length; i++) {
    print('\n[${i + 1}] State: ${states[i].runtimeType}');
    print('-' * 60);
    updateUI(states[i]);
  }

  print('\n' + '=' * 60);
  print('KEUNGGULAN SEALED CLASS:');
  print('=' * 60);
  print('✅ Exhaustive switch');
  print('✅ Type-safe');
  print('✅ Pattern matching');
  print('✅ Clear hierarchy');
  print('=' * 60);
}

Ringkasan

Ringkasan

Sealed: Hierarki tertutup & aman untuk pattern matching.

Base: Hanya bisa di-extend (tutup akses implement).

Final: Segel total (tidak bisa di-extend atau di-implement).

Quick Guide:

Sealed: rightarrow Untuk State atau Result terbatas.

Base: rightarrow Untuk Framework atau Base Class.

Final: rightarrow Untuk Utility atau Constant Class.

Bonus base Class Repository

Repository menggunakan base class agar hanya bisa diwariskan dengan extends, bukan diimplementasikan sembarangan.

Tujuan:

Mencegah implementasi yang tidak sesuai

Menjaga aturan penggunaan class tetap terkontrol

Ciri utama:

Hanya bisa di-extends

Tidak bisa di-implements di luar library

truktur method (CRUD) tetap konsisten

ntinya: base class dipakai untuk menjaga konsistensi CRUD dan memastikan hanya class turunan yang mengikuti aturan yang bisa digunakan.

Bonus Final Class Mathutils

MathUtils menggunakan final class karena hanya berisi fungsi utilitas (biasanya static) dan tidak perlu diwariskan.

Tujuan:

Mencegah class di-extend atau diimplementasikan

Menjaga fungsi tetap aman, stabil, dan tidak diubah subclass

Ciri utama:

Tidak bisa diwariskan

Cocok untuk method static

Kesimpulan: final class dipakai agar MathUtils tetap sederhana, aman, dan tidak bisa dimodifikasi oleh class lain.

// Nama: Ratu Klarisa Rahadianti
// NIM: 251410010
// Kelas: SI2A

// ================= Final Class MathUtils =================
final class MathUtils {
  // Penjumlahan
  static int tambah(int a, int b) {
    return a + b;
  }

  // Pengurangan
  static int kurang(int a, int b) {
    return a - b;
  }

  // Perkalian
  static int kali(int a, int b) {
    return a * b;
  }

  // Pembagian
  static double bagi(int a, int b) {
    if (b == 0) {
      throw Exception("Tidak bisa dibagi dengan nol");
    }
    return a / b;
  }
}

// ================= MAIN =================
void main() {
  print("Tambah: ${MathUtils.tambah(10, 5)}");
  print("Kurang: ${MathUtils.kurang(10, 5)}");
  print("Kali: ${MathUtils.kali(10, 5)}");
  print("Bagi: ${MathUtils.bagi(10, 5)}");
}
