📖 Rangkuman Materi (Slide 1–16)

  • 🔹 Polimorfisme : objek child dapat diperlakukan sebagai tipe parent (runtime via overriding).
  • 🔹 is & as : pengecekan tipe & casting aman.

  • 🔹 Static members : milik class, diakses via ClassName. Berguna untuk utility.

🎭 Pengantar Polimorfisme

Arti: "Banyak bentuk" (poly = banyak, morph = bentuk)

Konsep: Objek dari subclass dapat diperlakukan sebagai objek dari superclass

Contoh: Semua Kucing , Anjing , Burung adalah Hewan

Tipe:

i.Compile-time polymorphism: Method overloading (tidak didukung Dart)

ii.Runtime polymorphism: Method overriding (didukung Dart)

Referensi parent bisa menampung objek child. Method yang dipanggil sesuai objek asli (runtime).

🧮 5.MathUtils – Static Methods (Faktorial, Prima, Random, Rata2, Median)(Bonus 3)

//Nama:Ratu Klarisa
//Nim: 251410010
//Kelas:SI2A

import 'dart:math';

class MathUtils {
  // Faktorial
  static int faktorial(int n) {
    if (n <= 1) return 1;
    return n * faktorial(n - 1);
  }

  // Cek bilangan prima
  static bool isPrima(int n) {
    if (n < 2) return false;
    for (int i = 2; i <= sqrt(n); i++) {
      if (n % i == 0) return false;
    }
    return true;
  }

  // Membulatkan
  static double bulatkan(double angka, int digit) {
    return double.parse(angka.toStringAsFixed(digit));
  }

  // Random integer
  static int randomInt(int min, int max) {
    return min + Random().nextInt(max - min + 1);
  }

  // Rata-rata
  static double rataRata(List data) {
    return data.reduce((a, b) => a + b) / data.length;
  }

  // Median
  static double median(List data) {
    data.sort();
    int n = data.length;
    if (n % 2 == 0) {
      return (data[n ~/ 2 - 1] + data[n ~/ 2]) / 2;
    } else {
      return data[n ~/ 2].toDouble();
    }
  }
}

void main() {
  print(MathUtils.faktorial(5));
  print(MathUtils.isPrima(7));
  print(MathUtils.bulatkan(3.14159, 2));
  print(MathUtils.randomInt(1, 10));
  print(MathUtils.rataRata([10, 20, 30]));
  print(MathUtils.median([10, 20, 30, 40]));
}

Pengantar abstraksi

Konsep: Proses menyembunyikan detail teknis implementasi dan hanya menampilkan fungsi utamanya.

Tujuan:

i.Menekankan pada “apa yang dilakukan” bukan “bagaimana cara melakukannya”

ii.Menjadi aturan atau kontrak yang wajib diikuti

iii.Memberikan kebebasan dalam pengembangan implementasi

Cara di Dart: Menggunakan abstract class dan interface

🧱 Abstract Class

Abstract class adalah kelas yang tidak dapat dibuat objeknya secara langsung, melainkan digunakan sebagai dasar (parent) bagi kelas lain.

Kelas ini dapat berisi method abstract (tanpa isi) yang wajib diimplementasikan oleh subclass, serta method concrete (dengan isi) yang bisa

langsung digunakan.

Contoh di dart:

//Nama: Ratu Klarisa
//Nim: 251410010
//Kelas: SI2A


abstract class Kendaraan {
  String? merk; // Field

  void start(); // Method abstract

  void stop() { // Method concrete
    print('Kendaraan berhenti');
  }
}

class Mobil extends Kendaraan {
  @override
  void start() {
    print('Mobil $merk menyala');
  }
}

void main() {
  Mobil mobil = Mobil();
  mobil.merk = "Toyota";

  mobil.start();
  mobil.stop();
}

🧱 Abstract Method

Abstract method adalah method yang tidak memiliki isi (tanpa implementasi) di dalam abstract class dan harus

diimplementasikan (di-override) oleh subclass. Method ini berfungsi sebagai “kontrak” yang wajib diikuti oleh

setiap kelas turunan.

// Nama: Ratu Klarisa
// Nim: 251410010
// Kelas: SI2A

abstract class AlatMusik {
  void mainkan(); // Abstract method

  void info() {
    // Concrete method
    print('Ini adalah alat musik');
  }
}

class Gitar extends AlatMusik {
  @override
  void mainkan() {
    print('Jreng jreng...');
  }
}

void main() {
  Gitar gitar = Gitar();
  gitar.info();
  gitar.mainkan();
}

🛸 Interface (implements)

Interface di Dart (Sejak Dart 3.0)

Interface adalah struktur yang digunakan untuk mendefinisikan method tanpa implementasi sebagai

“kontrak” yang harus dipenuhi oleh class lain. Di Dart, setiap class secara otomatis dapat menjadi interface

(implicit interface), dan keyword interface digunakan untuk memperjelas peran tersebut.

//Nama: Ratu Klarisa
//Nim: 251410010
//Kelas: SI2A

      abstract class Flyable {
  void terbang();
}

class Bird implements Flyable {
  @override
  void terbang() {
    print('Burung terbang tinggi');
  }
}

class Plane implements Flyable {
  @override
  void terbang() {
    print('Pesawat terbang cepat');
  }
}

void main() {
  Bird burung = Bird();
  Plane pesawat = Plane();

  burung.terbang();
  pesawat.terbang();
}

📐 Static Members (MathUtils)

Static members adalah properti atau method yang dimiliki oleh class itu

sendiri, bukan oleh objek class. Static member diakses langsung melalui class name.

Properti dan method static sangat berguna untuk membuat utility class

atau menyimpan konstanta yang terkait dengan class, sehingga dapat diakses

langsung tanpa perlu membuat objek.

/Nama: Ratu Klarisa
//Nim: 251410010
//Kelas: SI2A

class MathUtils {
static double pi = 3.14;
static int add(int x, int y) {
return x + y;
}
}
void main() {
print('Nilai pi : ${MathUtils.pi}');
print('Result : ${MathUtils.add(10, 20)}');
}

}

🎵 Challenge: MediaPlayer + Playable/Pausable

Abstract MediaPlayer, interface Playable & Pausable, class AudioPlayer & VideoPlayer.
//Nama: Ratu Klarisa 
//Nim: 251410010
//Kelas: SI2A

// Abstract class
abstract class MediaPlayer {
  void info() {
    print('Media Player aktif');
  }
}

// Interface (pakai abstract class di Dart)
abstract class Playable {
  void play();
}

abstract class Pausable {
  void pause();
}

// Class implementasi
class AudioPlayer extends MediaPlayer implements Playable, Pausable {
  @override
  void play() {
    print('Memutar audio...');
  }

  @override
  void pause() {
    print('Audio dijeda');
  }
}

class VideoPlayer extends MediaPlayer implements Playable, Pausable {
  @override
  void play() {
    print('Memutar video...');
  }

  @override
  void pause() {
    print('Video dijeda');
  }
}

void main() {
  AudioPlayer audio = AudioPlayer();
  VideoPlayer video = VideoPlayer();

  audio.info();
  audio.play();
  audio.pause();

  print('---');

  video.info();
  video.play();
  video.pause();
}

🧱 Abstract Class

Abstract class adalah kelas yang tidak dapat dibuat objeknya secara langsung, melainkan digunakan sebagai dasar (parent) bagi kelas lain.

Kelas ini dapat berisi method abstract (tanpa isi) yang wajib diimplementasikan oleh subclass, serta method concrete (dengan isi) yang bisa

langsung digunakan.

Contoh di dart:

//Nama: Ratu Klarisa
//Nim: 251410010
//Kelas: SI2A


abstract class Kendaraan {
  String? merk; // Field

  void start(); // Method abstract

  void stop() { // Method concrete
    print('Kendaraan berhenti');
  }
}

class Mobil extends Kendaraan {
  @override
  void start() {
    print('Mobil $merk menyala');
  }
}

void main() {
  Mobil mobil = Mobil();
  mobil.merk = "Toyota";

  mobil.start();
  mobil.stop();
}

🧱 Abstract Method

Abstract method adalah method yang tidak memiliki isi (tanpa implementasi) di dalam abstract class dan harus

diimplementasikan (di-override) oleh subclass. Method ini berfungsi sebagai “kontrak” yang wajib diikuti oleh

setiap kelas turunan.

// Nama: Ratu Klarisa
// Nim: 251410010
// Kelas: SI2A

abstract class AlatMusik {
  void mainkan(); // Abstract method

  void info() {
    // Concrete method
    print('Ini adalah alat musik');
  }
}

class Gitar extends AlatMusik {
  @override
  void mainkan() {
    print('Jreng jreng...');
  }
}

void main() {
  Gitar gitar = Gitar();
  gitar.info();
  gitar.mainkan();
}

🛸 Interface (implements)

Interface di Dart (Sejak Dart 3.0)

Interface adalah struktur yang digunakan untuk mendefinisikan method tanpa implementasi sebagai

“kontrak” yang harus dipenuhi oleh class lain. Di Dart, setiap class secara otomatis dapat menjadi interface

(implicit interface), dan keyword interface digunakan untuk memperjelas peran tersebut.

//Nama: Ratu Klarisa
//Nim: 251410010
//Kelas: SI2A

      abstract class Flyable {
  void terbang();
}

class Bird implements Flyable {
  @override
  void terbang() {
    print('Burung terbang tinggi');
  }
}

class Plane implements Flyable {
  @override
  void terbang() {
    print('Pesawat terbang cepat');
  }
}

void main() {
  Bird burung = Bird();
  Plane pesawat = Plane();

  burung.terbang();
  pesawat.terbang();
}

📐 Static Members (MathUtils)

Static members adalah properti atau method yang dimiliki oleh class itu

sendiri, bukan oleh objek class. Static member diakses langsung melalui class name.

Properti dan method static sangat berguna untuk membuat utility class

atau menyimpan konstanta yang terkait dengan class, sehingga dapat diakses

langsung tanpa perlu membuat objek.

/Nama: Ratu Klarisa
//Nim: 251410010
//Kelas: SI2A

class MathUtils {
static double pi = 3.14;
static int add(int x, int y) {
return x + y;
}
}
void main() {
print('Nilai pi : ${MathUtils.pi}');
print('Result : ${MathUtils.add(10, 20)}');
}

Studi kasus: Sistem Bentuk Geometri

//Nama: Ratu Klarisa 
//Nim: 251410010
//Kelas: SI2A

abstract class Bentuk {
  double hitungLuas();
  double hitungKeliling();
  
  void info() {
    print('Luas: ${hitungLuas()}, Keliling: ${hitungKeliling()}');
  }
}

class Persegi extends Bentuk {
  double sisi;
  Persegi(this.sisi);
  
  @override
  double hitungLuas() => sisi * sisi;
  
  @override
  double hitungKeliling() => 4 * sisi;
}

class Lingkaran extends Bentuk {
  double radius;
  Lingkaran(this.radius);
  
  @override
  double hitungLuas() => 3.14 * radius * radius;
  
  @override
  double hitungKeliling() => 2 * 3.14 * radius;
}

void main() {
  Persegi p = Persegi(4);
  Lingkaran l = Lingkaran(7);

  p.info();
  l.info();
}

🎵 Challenge: MediaPlayer + Playable/Pausable

Abstract MediaPlayer, interface Playable & Pausable, class AudioPlayer & VideoPlayer.
//Nama: Ratu Klarisa 
//Nim: 251410010
//Kelas: SI2A

// Abstract class
abstract class MediaPlayer {
  void info() {
    print('Media Player aktif');
  }
}

// Interface (pakai abstract class di Dart)
abstract class Playable {
  void play();
}

abstract class Pausable {
  void pause();
}

// Class implementasi
class AudioPlayer extends MediaPlayer implements Playable, Pausable {
  @override
  void play() {
    print('Memutar audio...');
  }

  @override
  void pause() {
    print('Audio dijeda');
  }
}

class VideoPlayer extends MediaPlayer implements Playable, Pausable {
  @override
  void play() {
    print('Memutar video...');
  }

  @override
  void pause() {
    print('Video dijeda');
  }
}

void main() {
  AudioPlayer audio = AudioPlayer();
  VideoPlayer video = VideoPlayer();

  audio.info();
  audio.play();
  audio.pause();

  print('---');

  video.info();
  video.play();
  video.pause();
}

  • ✔ Static members berguna untuk fungsi utilitas tanpa instansiasi objek.
  • ✔ Kombinasi semua konsep menciptakan arsitektur yang modular dan scalable.
